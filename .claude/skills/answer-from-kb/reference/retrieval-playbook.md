# Retrieval playbook — broad, parallel, file-based

The corpus is ~620 talks. One narrow grep will miss things. **Fan out several lenses in parallel
and bias toward recall.** No vector DB — it's all `rg`/`Read` over Markdown, plus the nav tree and
`kb/graph.json`.

## The nav tree (start here, drill down)

```
kb/INDEX.md                     # topics (with counts), people, series — the map
  kb/topics/<t>.md              # synthesis + "Browse all N talks tagged <t>" (backlinks to talks)
  kb/series/<s>.md              # every talk in a series, by year
  kb/entities/people/<p>.md     # positions + "Browse all N talks featuring <p>"
    kb/talks/<slug>.md          # thesis + timestamped, cited key moments
kb/graph.json                   # typed edges: about / authored-by / part-of / links-to / contradicts
```
Follow `[[wikilinks]]` and `See also`. To enumerate the talks behind a topic/person, read the
"Browse all N talks…" block on its page (generated by `scripts/build_nav.py`).

## The five lenses (run in parallel; pick 3–6)

| Lens | What it does |
|---|---|
| **Topic** | Open the 2–4 most relevant `kb/topics/*`; read their synthesis + backlinked talks. |
| **Keyword** | Broad `rg -i` across all `kb/` with an expanded OR-pattern (synonyms/acronyms/jargon). |
| **Reframe** | Re-ask in 2–3 vocabularies (beginner / operator / opposite) and search each. |
| **People** | Find likely speakers in `kb/entities/people/`; read their pages + browse their talks. |
| **Adjacency** | From the top topics, follow `See also` + `graph.json` to *adjacent* topics/talks. |

## Query expansion (extend freely — recall first)

| Asked about | OR-pattern |
|---|---|
| enterprise / B2B sales | `enterprise\|B2B\|sales\|procurement\|champion\|pilot\|POC\|stall\|urgency\|contract\|forward.deployed` |
| product-market fit | `product.?market fit\|PMF\|retention\|pull\|desperate\|must.?have\|cohort\|sucking sound` |
| pricing | `pricing\|price\|charge\|willingness to pay\|discount\|anchor\|tier\|usage.?based\|value equation` |
| fundraising | `raise\|fundrais\|valuation\|SAFE\|dilution\|term sheet\|angel\|Series A\|runway\|default alive` |
| hiring / team | `hire\|hiring\|recruit\|co.?founder\|equity\|first employee\|firing\|renaissance` |
| growth | `growth\|acquisition\|channel\|virality\|retention\|funnel\|CAC\|LTV\|word of mouth` |
| ideas | `startup idea\|tinker\|curiosity\|pivot\|second mover\|live in the future` |

## `rg` recipes

```bash
rg -i -l 'kw1|kw2|kw3' kb/ -g '*.md'          # route: which pages match
rg -i -C 3 'kw1|kw2' kb/topics/ kb/talks/     # citable passages with context
rg -i -c 'kw' kb/talks/*.md | sort -t: -k2 -rn  # cheap per-file relevance score
python3 scripts/search.py search "how to price a B2B product"   # ranked, offline
```

## Rules
- **Recall > precision.** Include a maybe-relevant talk; synthesis filters later.
- **Read the pages you shortlist** — don't cite from filenames.
- **Reuse the citations already on the pages** — never invent a timestamp or quote.
- **Keep contradictions** — they're the most valuable output; cite both sides.
