# Should I Use a Dev Shop? - Michael Seibel — Compiled Truth

Michael Seibel warns that outsourcing your first product build to a dev shop instead of finding a technical co-founder is a common trap that looks like progress but usually costs more money, more time, and hurts fundraising.

**Key ideas (with timestamps):**
- Non-technical founders are commonly advised to use a dev shop to build an initial version, get traction, then hire a CTO later [should-i-use-a-dev-shop-michael-seibel @ 00:00](https://youtu.be/QAJQOXMM7ak?t=0)
- This advice isn't inherently wrong, but survivorship bias hides the many founders for whom the dev shop path has failed [should-i-use-a-dev-shop-michael-seibel @ 00:48](https://youtu.be/QAJQOXMM7ak?t=48)
- Building an MVP through an outsourced dev shop typically takes longer than expected because the shop is juggling other projects and isn't as motivated as an equity-holding founder [should-i-use-a-dev-shop-michael-seibel @ 01:09](https://youtu.be/QAJQOXMM7ak?t=69)
- Costs commonly balloon well beyond the initial fifteen to thirty thousand dollar estimate, which is especially painful for founders spending limited angel money [should-i-use-a-dev-shop-michael-seibel @ 01:26](https://youtu.be/QAJQOXMM7ak?t=86)
- Many good tech startup investors won't invest in companies that outsourced their development unless growth is already extraordinary [should-i-use-a-dev-shop-michael-seibel @ 01:46](https://youtu.be/QAJQOXMM7ak?t=106)
- Work built by an outsourced dev shop is often thrown away and rebuilt once a founder finally hires a developer who disagrees with the original approach [should-i-use-a-dev-shop-michael-seibel @ 02:09](https://youtu.be/QAJQOXMM7ak?t=129)
- The trap doesn't look like a trap at first because things finally start being built, even as money and time drain away [should-i-use-a-dev-shop-michael-seibel @ 03:08](https://youtu.be/QAJQOXMM7ak?t=188)
- Good early-stage investors care less about traction and more about whether the founders can build and iterate quickly, which the dev shop route makes slower and more expensive [should-i-use-a-dev-shop-michael-seibel @ 04:03](https://youtu.be/QAJQOXMM7ak?t=243)
- Recruiting a technical co-founder instead typically costs less, ships product faster, and sets founders up to iterate quickly on customer feedback [should-i-use-a-dev-shop-michael-seibel @ 04:45](https://youtu.be/QAJQOXMM7ak?t=285)
- Founders often give up on finding a technical co-founder too easily and default to a dev shop, when more effort recruiting a co-founder is usually worth it [should-i-use-a-dev-shop-michael-seibel @ 05:06](https://youtu.be/QAJQOXMM7ak?t=306)
