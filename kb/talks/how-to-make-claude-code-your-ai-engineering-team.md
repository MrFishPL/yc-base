# How to Make Claude Code Your AI Engineering Team — Compiled Truth

Garry Tan argues the bottleneck in AI coding isn't model intelligence but scaffolding, and demonstrates G-stack, his open-source "thin harness, fat skills" system that turns Claude Code into a team of specialist skills modeled on YC's own process.

**Key ideas (with timestamps):**
- Out of the box, models wander and guess at your codebase because the scaffolding around them is too thin, not because the model lacks intelligence [how-to-make-claude-code-your-ai-engineering-team @ 01:39](https://youtu.be/wkv2ifxPpF8?t=99)
- G-stack is a "thin harness, fat skills" open-source system that turns Claude Code into an AI engineering team with skills that act like specialists [how-to-make-claude-code-your-ai-engineering-team @ 02:03](https://youtu.be/wkv2ifxPpF8?t=123)
- The "office hours" skill is modeled directly on real YC partner office hours and opens with forcing questions like "what's the strongest evidence that someone actually wants this?" before any code is written [how-to-make-claude-code-your-ai-engineering-team @ 05:16](https://youtu.be/wkv2ifxPpF8?t=316)
- Office hours pushes back on weak premises and helps reframe a basic idea (e.g., a document aggregator) into a bigger wedge strategy (aggregator as lead-gen funnel into a higher-margin service) [how-to-make-claude-code-your-ai-engineering-team @ 06:19](https://youtu.be/wkv2ifxPpF8?t=379)
- About one in three times a founder runs office hours, the honest conclusion is that the idea doesn't make sense, which Tan considers a feature, not a failure [how-to-make-claude-code-your-ai-engineering-team @ 12:15](https://youtu.be/wkv2ifxPpF8?t=735)
- A multi-step adversarial review skill stress-tests the design doc, automatically found and fixed 16 issues, and raised the doc's score from 6/10 to 8/10 [how-to-make-claude-code-your-ai-engineering-team @ 12:53](https://youtu.be/wkv2ifxPpF8?t=773)
- "Design shotgun" generates multiple AI visual design directions in about 60 seconds so founders can pick a favorite rather than design from scratch [how-to-make-claude-code-your-ai-engineering-team @ 13:56](https://youtu.be/wkv2ifxPpF8?t=836)
- Users of G-stack report spending 80-90% of their time in the office-hours, CEO-review, and auto-plan skills rather than in raw coding [how-to-make-claude-code-your-ai-engineering-team @ 16:19](https://youtu.be/wkv2ifxPpF8?t=979)
- Tan runs 10-15 parallel Claude Code sessions across multiple open-source projects, sometimes landing 10-50 PRs in a day, using worktrees created per idea or bug report [how-to-make-claude-code-your-ai-engineering-team @ 19:36](https://youtu.be/wkv2ifxPpF8?t=1176)
- A custom CLI wrapping Playwright/Chromium (slash-QA, slash-browse) was built because the Claude in Chrome MCP was slow and unreliable, letting agents take screenshots, click, fill forms, and run regression tests directly [how-to-make-claude-code-your-ai-engineering-team @ 17:24](https://youtu.be/wkv2ifxPpF8?t=1044)
