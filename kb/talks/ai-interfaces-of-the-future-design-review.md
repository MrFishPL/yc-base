# AI Interfaces Of The Future | Design Review — Compiled Truth

In this design-review episode, the host and Notion Calendar creator Raphael Schaad critique a batch of YC-community AI products to surface recurring interface patterns for the post-chat-UI era, where software increasingly has to represent "verbs" (workflows, agentic actions) instead of static nouns.

**Key ideas (with timestamps):**
- Traditional software interfaces are built around static "nouns" (buttons, forms, dropdowns), but AI-native interfaces need to represent "verbs" — workflows, autocomplete, and autonomous actions — for which good tooling and conventions don't fully exist yet [ai-interfaces-of-the-future-design-review @ 01:01](https://youtu.be/DBhSfROq3wU?t=61)
- Voice AI demos (Vapi, Retail AI) show that latency is effectively the interface: even a small delay is what breaks the illusion of talking to a human, and exposing millisecond-level latency in a "dev mode" helps developers build intuition for what feels natural [ai-interfaces-of-the-future-design-review @ 02:58](https://youtu.be/DBhSfROq3wU?t=178)
- Visual canvas/flowchart interfaces (Gumloop) are emerging as the natural way to model branching, multi-step agent decision trees, since linear documents can't capture the multi-dimensionality of agentic workflows [ai-interfaces-of-the-future-design-review @ 11:12](https://youtu.be/DBhSfROq3wU?t=672)
- Turning example prompts into single-click buttons (Answer Grid) helps solve the "blank canvas" problem where users don't know what to type into an open-ended AI prompt box [ai-interfaces-of-the-future-design-review @ 13:11](https://youtu.be/DBhSfROq3wU?t=791)
- Inline citations/footnotes attached to AI-generated answers (popularized by Perplexity) let users validate and trust agent-retrieved data by clicking through to the original source [ai-interfaces-of-the-future-design-review @ 17:14](https://youtu.be/DBhSfROq3wU?t=1034)
- For prompt-to-output tools with long generation times (Polymet), keeping users engaged during multi-minute waits and giving feedback on what the model did or didn't respect from the prompt is a key unsolved design challenge [ai-interfaces-of-the-future-design-review @ 20:32](https://youtu.be/DBhSfROq3wU?t=1232)
- Adaptive interfaces that generate UI dynamically based on content (Zeno's email app) can replace generic buttons with content-specific actions, using single-letter hotkeys since users are already hands-on-keyboard [ai-interfaces-of-the-future-design-review @ 29:02](https://youtu.be/DBhSfROq3wU?t=1742)
- AI video generation (Argil) trades fidelity for immediacy by first showing a blurry, fast audio-matched preview before committing to a slow, high-fidelity full render, keeping the human in an iterative feedback loop [ai-interfaces-of-the-future-design-review @ 32:52](https://youtu.be/DBhSfROq3wU?t=1972)
- The overall shift from single chat-box interfaces to a proliferation of AI-native components in just a year or two is compared to the touch-first reinvention of software interfaces around 2010 [ai-interfaces-of-the-future-design-review @ 35:35](https://youtu.be/DBhSfROq3wU?t=2135)
