# Vibe Coding Is The Future — Compiled Truth

YC partners survey the current batch on "vibe coding" and conclude it is not a fad but a fundamental, accelerating shift in how software gets built and how engineers should be hired and evaluated.

**Key ideas (with timestamps):**
- One quarter of surveyed founders said more than 95% of their codebase is AI-generated, even among highly technical founders who could have built it by hand [vibe-coding-is-the-future @ 09:51](https://youtu.be/IACHfKmZMr8?t=591)
- Founders describe a role shift from "software engineer" to "product engineer," where human taste matters more than raw coding ability as AI makes everyone a 10x engineer [vibe-coding-is-the-future @ 01:09](https://youtu.be/IACHfKmZMr8?t=69)
- Current AI coding tools are notably bad at debugging, so humans still must dig into the code to find logic errors [vibe-coding-is-the-future @ 04:19](https://youtu.be/IACHfKmZMr8?t=259)
- Because rewriting code is now nearly free, founders' workflow has shifted to "reroll and rewrite from scratch" rather than carefully fixing bugs, similar to rerolling AI image generations [vibe-coding-is-the-future @ 04:52](https://youtu.be/IACHfKmZMr8?t=292)
- Cursor remains the dominant coding tool in the batch, but Windsurf is a fast follower because it indexes the whole codebase automatically rather than requiring users to point it at specific files [vibe-coding-is-the-future @ 07:02](https://youtu.be/IACHfKmZMr8?t=422)
- Founders still turn to reasoning models like o1/o3 and ChatGPT for hard debugging, since Cursor/Windsurf are described as still "pre-reasoning" tools [vibe-coding-is-the-future @ 08:12](https://youtu.be/IACHfKmZMr8?t=492)
- Historically, companies like Stripe and Gusto succeeded by hiring for raw coding productivity over classical CS training, shifting interviews from "teach us how you think" to "build a to-do app as fast as you can" [vibe-coding-is-the-future @ 13:05](https://youtu.be/IACHfKmZMr8?t=785)
- There is a recurring "zero-to-one vs one-to-n" split: vibe coding and lightweight frameworks get founders to product-market fit fast, but scaling (e.g., Twitter's fail whale, Facebook's HipHop compiler) still requires classically-trained systems engineers [vibe-coding-is-the-future @ 17:11](https://youtu.be/IACHfKmZMr8?t=1031)
- The panel argues technical hiring should now explicitly screen for how well candidates use AI coding tools, since old interview questions can simply be pasted into ChatGPT for a perfect answer [vibe-coding-is-the-future @ 20:13](https://youtu.be/IACHfKmZMr8?t=1213)
- Being technical enough to catch AI agents (or human employees) lying or cutting corners is described as a superpower founders still need, illustrated by a story about engineers falsely claiming a feature "can't be done" [vibe-coding-is-the-future @ 29:05](https://youtu.be/IACHfKmZMr8?t=1745)
