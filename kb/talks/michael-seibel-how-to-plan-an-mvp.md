# Michael Seibel - How to Plan an MVP — Compiled Truth

Michael Seibel argues an MVP should be the ridiculously simple, fast-to-build first thing you give real users to test whether you can deliver any value, and founders should launch it quickly rather than chase a polished "press launch."

**Key ideas (with timestamps):**
- An MVP is simply the first thing you can give your very first target users to see if you can deliver any value at all [michael-seibel-how-to-plan-an-mvp @ 00:21](https://youtu.be/1hHMwLxN6EM?t=21)
- Get your first users by talking to the specific person you already know has the problem, rather than searching for a mysterious audience [michael-seibel-how-to-plan-an-mvp @ 01:18](https://youtu.be/1hHMwLxN6EM?t=78)
- The core early-stage sequence is: launch quickly, get any initial customers, talk to them for feedback, and iterate [michael-seibel-how-to-plan-an-mvp @ 02:04](https://youtu.be/1hHMwLxN6EM?t=124)
- Hold the problem you're solving and the customer tightly, but hold the specific solution loosely since it may not be what customers actually want [michael-seibel-how-to-plan-an-mvp @ 03:23](https://youtu.be/1hHMwLxN6EM?t=203)
- Iterating means fixing the broken solution while keeping the same problem and customer, rather than pivoting the "screwdriver" to solve unrelated problems [michael-seibel-how-to-plan-an-mvp @ 03:43](https://youtu.be/1hHMwLxN6EM?t=223)
- A lean MVP should be buildable in weeks, not months, with extremely limited functionality focused on a small set of initial users [michael-seibel-how-to-plan-an-mvp @ 04:17](https://youtu.be/1hHMwLxN6EM?t=257)
- Airbnb, Twitch, and Stripe all launched with strikingly bare-bones first versions (no payments, one channel, no bank deals) yet became billion-dollar companies [michael-seibel-how-to-plan-an-mvp @ 05:27](https://youtu.be/1hHMwLxN6EM?t=327)
- Heavy MVPs (regulated industries, hard tech, biotech) can still start as a simple explanatory website while the real product takes longer to build [michael-seibel-how-to-plan-an-mvp @ 08:18](https://youtu.be/1hHMwLxN6EM?t=498)
- Founders should separate "launch" (getting any customers) from "press launch" (getting buzz), and prioritize the former since nobody remembers when Google, Facebook, or Twitter actually launched [michael-seibel-how-to-plan-an-mvp @ 09:23](https://youtu.be/1hHMwLxN6EM?t=563)
- To ship fast, time-box your spec to what's buildable in a set window, write the spec down so you notice scope creep, and cut features when you're behind [michael-seibel-how-to-plan-an-mvp @ 11:18](https://youtu.be/1hHMwLxN6EM?t=678)
- Don't fall in love with your MVP; it's just a starting point to iterate from, not the final vision [michael-seibel-how-to-plan-an-mvp @ 12:49](https://youtu.be/1hHMwLxN6EM?t=769)
