# Andrej Karpathy: Software Is Changing (Again) — Compiled Truth

Karpathy argues that software is undergoing its second fundamental shift in a few years — into "Software 3.0" where LLMs are programmed in plain English — and that builders should design fallible-but-superhuman LLMs into partial-autonomy products with fast generation/verification loops rather than fully autonomous agents.

**Key ideas (with timestamps):**
- Software 1.0 is written code, Software 2.0 is neural net weights, and Software 3.0 is LLMs programmed via English-language prompts — a genuinely new, third programming paradigm [andrej-karpathy-software-is-changing-again @ 02:50](https://youtu.be/LCEmiRjPEtQ?t=170)
- LLMs behave like a strange hybrid of a utility, a fab, and — most usefully — an early-1960s-style operating system, still centralized and time-shared because per-user compute is too expensive for "personal computing" [andrej-karpathy-software-is-changing-again @ 09:19](https://youtu.be/LCEmiRjPEtQ?t=559)
- Unlike past transformative technologies, LLM adoption is flipped: consumers got access before governments and militaries, an unprecedented diffusion pattern [andrej-karpathy-software-is-changing-again @ 13:05](https://youtu.be/LCEmiRjPEtQ?t=785)
- LLMs are best understood psychologically as fallible "people spirits" with encyclopedic memory but real cognitive deficits — hallucination, jagged intelligence, no persistent memory across sessions, and susceptibility to prompt injection [andrej-karpathy-software-is-changing-again @ 14:50](https://youtu.be/LCEmiRjPEtQ?t=890)
- The best current LLM apps (e.g., Cursor, Perplexity) share key traits: heavy context management, orchestration of multiple model calls, a custom GUI for fast human auditing, and an adjustable "autonomy slider" [andrej-karpathy-software-is-changing-again @ 19:12](https://youtu.be/LCEmiRjPEtQ?t=1152)
- Because humans remain the bottleneck on verifying AI output, builders should keep the "AI on a leash" — favor small, concrete, incremental diffs over giant autonomous changes — to keep the generation/verification loop fast [andrej-karpathy-software-is-changing-again @ 22:20](https://youtu.be/LCEmiRjPEtQ?t=1340)
- Full autonomy is much harder than demos suggest — Karpathy's 2013 flawless Waymo ride felt like self-driving was imminent, yet 12 years later autonomy still isn't fully solved, so expect this to be "the decade of agents," not the year [andrej-karpathy-software-is-changing-again @ 26:13](https://youtu.be/LCEmiRjPEtQ?t=1573)
- "Vibe coding" — describing intent in English and letting an LLM write the code — is opening programming to non-programmers, though the coding itself is often the easy part versus auth, payments, and deployment plumbing [andrej-karpathy-software-is-changing-again @ 29:24](https://youtu.be/LCEmiRjPEtQ?t=1764)
- Builders should also design for AI agents as a new class of consumer of digital information — via markdown-based docs, llm.txt files, replacing "click here" instructions with executable commands, and protocols like MCP [andrej-karpathy-software-is-changing-again @ 34:00](https://youtu.be/LCEmiRjPEtQ?t=2040)
