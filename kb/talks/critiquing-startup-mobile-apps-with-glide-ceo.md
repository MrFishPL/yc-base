# Critiquing Startup Mobile Apps with Glide CEO — Compiled Truth

Glide co-founder David Siegel joins a design review to critique founder-submitted mobile apps, highlighting recurring mistakes in tap-target sizing, color consistency, onboarding length, and interaction clarity.

**Key ideas (with timestamps):**
- Well-designed mobile apps should stick to platform standards and account for the physical context in which users hold and use their phone [critiquing-startup-mobile-apps-with-glide-ceo @ 01:03](https://youtu.be/4N55fVuxxOg?t=63)
- Tappable elements on mobile should generally be larger than 60 pixels square since anything smaller is hard to hit accurately with a thumb [critiquing-startup-mobile-apps-with-glide-ceo @ 02:31](https://youtu.be/4N55fVuxxOg?t=151)
- If a product's name references a color (like "Blue Dot"), the brand's primary visual color should match that name rather than contradict it [critiquing-startup-mobile-apps-with-glide-ceo @ 06:43](https://youtu.be/4N55fVuxxOg?t=403)
- Two visually similar CTA buttons with different priority (like a primary "start charging" and secondary "route") should be differentiated by giving the secondary action a neutral color [critiquing-startup-mobile-apps-with-glide-ceo @ 08:18](https://youtu.be/4N55fVuxxOg?t=498)
- The best way to catch UX friction is to get the app into a real user's hands as quickly as possible and write down exactly where they trip up, rather than guessing at fixes [critiquing-startup-mobile-apps-with-glide-ceo @ 09:12](https://youtu.be/4N55fVuxxOg?t=552)
- Asking users too many onboarding questions before they reach value risks losing them, even though that data feels valuable to collect [critiquing-startup-mobile-apps-with-glide-ceo @ 15:46](https://youtu.be/4N55fVuxxOg?t=946)
- If a call-to-action previews specific content (like food items), tapping it should show the user that exact content rather than something disorienting and different [critiquing-startup-mobile-apps-with-glide-ceo @ 11:49](https://youtu.be/4N55fVuxxOg?t=709)
- Gamification and upsell prompts (like rewards programs) should come after a user's first successful core action, not interrupt it and create "side quests" away from the main task [critiquing-startup-mobile-apps-with-glide-ceo @ 13:05](https://youtu.be/4N55fVuxxOg?t=785)
- An app that is mostly informational with no unique mobile-only interactions (like using location or profiles) is often better off as a website than a native app [critiquing-startup-mobile-apps-with-glide-ceo @ 22:15](https://youtu.be/4N55fVuxxOg?t=1335)
- Every user tap should get an immediate response, since responding late feels almost as bad as not responding at all [critiquing-startup-mobile-apps-with-glide-ceo @ 22:59](https://youtu.be/4N55fVuxxOg?t=1379)
