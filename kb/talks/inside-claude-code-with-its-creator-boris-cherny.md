# Inside Claude Code With Its Creator Boris Cherny — Compiled Truth

Boris Cherny, creator of Claude Code, explains how the product emerged almost entirely from watching what the model and users naturally wanted to do, and argues founders should build for the model six months from now rather than the model of today.

**Key ideas (with timestamps):**
- Build for the model six months from now, not the model of today, since the frontier keeps moving and whatever it can't do yet it will soon do [inside-claude-code-with-its-creator-boris-cherny @ 00:00](https://youtu.be/PQU9o_5rHC4?t=0)
- Claude Code started as an accidental side project — a simple terminal chat app Cherny built just to learn Anthropic's API and tool use, with no mandate to build a CLI [inside-claude-code-with-its-creator-boris-cherny @ 03:11](https://youtu.be/PQU9o_5rHC4?t=191)
- The single biggest product principle is latent demand: you can't get people to do a new thing, you can only make an existing behavior easier, which is how CLAUDE.md and plan mode were discovered from users' own workarounds [inside-claude-code-with-its-creator-boris-cherny @ 07:47](https://youtu.be/PQU9o_5rHC4?t=467)
- Scaffolding built around a model typically buys only a temporary 10-20% performance gain before the next model release wipes out the advantage, so teams must choose between building scaffolding or waiting for the model [inside-claude-code-with-its-creator-boris-cherny @ 08:04](https://youtu.be/PQU9o_5rHC4?t=484)
- CLAUDE.md files should stay minimal — if one balloons to thousands of tokens, delete it and rebuild only the instructions actually needed, since required scaffolding shrinks with every model generation [inside-claude-code-with-its-creator-boris-cherny @ 10:01](https://youtu.be/PQU9o_5rHC4?t=601)
- When hiring, Cherny screens for scientific, first-principles thinkers by asking about a time they were wrong, since strong prior opinions from senior engineers are often stale in a fast-moving model landscape [inside-claude-code-with-its-creator-boris-cherny @ 15:54](https://youtu.be/PQU9o_5rHC4?t=954)
- The most effective engineers on the Claude Code team are bimodal: extreme specialists in dev tools/runtimes, or hyper-generalists who span product, design, and research [inside-claude-code-with-its-creator-boris-cherny @ 18:43](https://youtu.be/PQU9o_5rHC4?t=1123)
- Anthropic's advice for dev-tool founders is to figure out what the model itself "wants" to do (e.g., use tools, hit APIs not websites) and build product around enabling that, the same way you'd build for a human user [inside-claude-code-with-its-creator-boris-cherny @ 31:10](https://youtu.be/PQU9o_5rHC4?t=1870)
- Since Claude Code launched, productivity per engineer at Anthropic has grown roughly 150%, measured via pull requests and commit activity, even as headcount doubled [inside-claude-code-with-its-creator-boris-cherny @ 40:39](https://youtu.be/PQU9o_5rHC4?t=2439)
- Cherny predicts coding will become generally solved and the "software engineer" title will fade into broader builder/product roles as every function on a team learns to code with agents [inside-claude-code-with-its-creator-boris-cherny @ 44:42](https://youtu.be/PQU9o_5rHC4?t=2682)
