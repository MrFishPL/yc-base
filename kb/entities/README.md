# kb/entities/

**What goes here:** one page per real-world entity.
- `people/<slug>.md` — founders, YC partners, guests (`type: person`).
- `companies/<slug>.md` — companies / products used as case studies (`type: company`).

Two-layer page: who they are + recurring positions (cited) above `---`; dated + cited Timeline of
talks/mentions below. **One page per entity** — all name variants go in `aliases:` (kills the
"PG" vs "Paul Graham" split-brain). Filename = identity.

**What does NOT go here:** talk summaries (→ `kb/talks/`) or topic synthesis (→ `kb/topics/`).
